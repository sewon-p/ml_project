# FCD 기반 교통류 변수 추정 결과 보고서

## 1. 개요

### 1.1 연구 목적

본 연구는 **프로브 차량 1대의 FCD(Floating Car Data)**만을 이용하여 도로 구간의 거시적 교통류 변수인 **밀도(k, veh/km)**와 **교통량(q, veh/hr)**을 추정하는 것을 목표로 한다.

### 1.2 접근 방법

교통류 기본도(Fundamental Diagram, FD)의 Underwood 지수 모델을 초기 추정값으로 사용하고, 그 잔차(실제값 - FD 추정값)를 머신러닝 모델이 보정하는 **하이브리드 구조**를 채택하였다.

```
최종 예측 = FD 초기 추정(k_fd) + ML 잔차 보정(delta_k)
```

### 1.3 실험 환경

- **시뮬레이터**: SUMO (Simulation of Urban Mobility)
- **시나리오**: 1~3차로 도로, 제한속도 50/60/80/100 km/h, 수요량 200~1400 veh/hr
- **FCD**: 프로브 1대의 300초간 궤적 (위치, 속도, 가속도)
- **Ground Truth**: 루프 검지기 기반 밀도, 교통량 (5분 집계)
- **데이터 분할**: 시나리오 ID 기반 80/20 split (데이터 누수 방지)

### 1.4 비교 모델

| 모델 | 유형 | 입력 |
|---|---|---|
| FD Underwood | 이론 모델 | 프로브 평균속도, 제한속도, 차선수 |
| CNN1D | 딥러닝 (1D Conv) | 시계열 6채널 + 환경변수 2개 |
| LSTM | 딥러닝 (순환신경망) | 시계열 6채널 + 환경변수 2개 |
| XGBoost | 앙상블 (Gradient Boosting) | 통계 피처 19개 (속도, 가속도, 정지비율 등) |

---

## 2. 전체 모델 성능 비교

### 2.1 Density (밀도, veh/km)

| 모델 | RMSE | MAE | MAPE | R2 |
|---|---|---|---|---|
| FD only | 23.61 | 18.51 | 211.9% | -65.3 |
| CNN1D | 6.97 | 4.32 | 46.7% | -4.79 |
| LSTM | 6.13 | 3.93 | 42.3% | -3.47 |
| **XGBoost** | **1.50** | **1.10** | **11.5%** | **0.731** |

### 2.2 Flow (교통량, veh/hr)

| 모델 | RMSE | MAE | MAPE | R2 |
|---|---|---|---|---|
| FD only | 1320.9 | 1056.1 | 205.0% | -138.5 |
| CNN1D | 665.8 | 460.8 | 88.8% | -34.4 |
| LSTM | 584.4 | 391.1 | 74.4% | -26.3 |
| **XGBoost** | **80.4** | **61.7** | **11.9%** | **0.484** |

> [!NOTE]
> Flow의 R2가 density보다 낮은 이유는 변수 스케일 차이이다 (veh/km vs veh/hr). MAPE 기준으로는 11.5% vs 11.9%로 거의 동등한 정확도를 보인다.

## 3. ML 성능 비교 분석

### 3.1 XGBoost가 최적 모델인 이유

Density와 Flow 모두에서 **XGBoost가 압도적 1위**를 기록하였다. 그 원인은 입력 피처의 정보량 차이에 있다.

- **XGBoost**: `speed_mean`, `speed_std`, `accel_std`, `brake_ratio`, `num_lanes`, `speed_limit` 등 **19개 통계 피처**를 사용하여, 프로브 궤적의 다양한 측면을 수치로 포착한다.
- **DL 모델**: 원시 시계열 6채널 + 환경 변수 2개만 사용한다. 시계열에서 유의미한 패턴을 자동 추출해야 하지만, 300초 × 6채널의 정보량으로는 hand-crafted 피처를 능가하지 못했다.

## 4. v-k Diagram: 차선별 상세 분석

XGBoost-Density 조합이 최적임을 확인한 후, FD와 ML의 예측 패턴을 차선별로 상세 비교한다.

![v-k Diagram 6패널. 상단: FD 예측(속도제한별 4색), 하단: XGBoost 예측(파랑) vs 실제(검정 테두리)](C:/Users/박세원/.gemini/antigravity/brain/8d5f44d7-1c61-4e10-97b3-d628e8698d13/graph1_fd_vs_ml_6panel.png)

### 4.1 FD Underwood의 한계 (상단 3패널)

- 속도제한별로 실제 데이터 점을 4색(파랑=50, 초록=60, 주황=80, 빨강=100 km/h)으로 구분하고, 대응하는 FD 이론 곡선을 같은 색으로 그렸다.
- **FD 곡선과 실제 데이터가 크게 불일치**: 곡선은 급격한 감소를 보이지만, 실제 데이터는 훨씬 완만하게 분포한다.
- 차선 수가 증가할수록 FD 오차가 심화된다 (1L: R2=-43.9, 3L: R2=-103.6).
- 원인: 프로브 1대의 속도는 도로 전체의 공간평균속도를 대표하지 못한다.

### 4.2 XGBoost 보정 효과 (하단 3패널)

- 검정 테두리 점(실제)과 파란 점(XGBoost 예측)이 거의 겹친다.
- 차선 수에 관계없이 안정적인 성능을 보인다 (R2 = 0.72 ~ 0.73).
- ML이 FD의 구조적 한계를 성공적으로 보정한다.

## 5. 결론

1. **FD(Underwood) 단독 사용은 불가능하다.** R2가 음수(=-65 ~ -138)로, 평균값 예측보다 못하다.
2. **XGBoost + FD 잔차 보정**이 최적 조합이다. Density MAPE **11.5%**, Flow MAPE **11.9%**로 실용적 정확도를 달성하였다.
3. **DL 모델(CNN1D, LSTM)은 XGBoost에 크게 못 미친다.** 환경 변수를 추가한 후에도 RMSE가 4배 이상 높다. 통계 피처의 정보량이 원시 시계열보다 이 과제에 더 적합하다.
4. **Density 추정이 Flow보다 안정적이다.** R2 기준 0.731 vs 0.484. Flow는 q = k x v의 곱셈 구조로 오차가 증폭된다.
5. **XGBoost의 차선별 성능이 균일하다** (1L: 1.39, 2L: 1.55, 3L: 1.56 RMSE). 도로 환경에 강건한 모델임을 시사한다.
